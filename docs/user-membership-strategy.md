## 四、存量用户处理策略

### 4.1 核心原则

**数据不删，只限新增**

### 4.2 处理规则

| 场景                             | 本地已有数据 | 处理方式                           |
| -------------------------------- | ------------ | ---------------------------------- |
| 老用户未登录，计数器>3个         | 保留全部     | 可正常使用，禁止新增，提示登录     |
| 老用户未登录，图解>10个          | 保留全部     | 可正常使用，禁止新增，提示登录     |
| 老用户已登录非会员，计数器>3个   | 保留并同步   | 可正常使用，禁止新增，提示开通会员 |
| 老用户已登录非会员，图解>10个    | 保留（不同步）| 可正常使用，禁止新增，提示开通会员 |

### 4.3 老用户判断逻辑

```typescript
function isLegacyUser(): boolean {
  const counterCount = getCounterKeys().length;
  const diagramCount = getImageList().length + getFileList().length;
  return counterCount > 3 || diagramCount > 10;
}
```

## 五、数据同步时机

采用混合同步策略，兼顾用户体验与数据安全：

| 触发场景       | 同步策略                 | 说明                                     |
| -------------- | ------------------------ | ---------------------------------------- |
| 创建/删除计数器 | 实时同步                 | 重要操作，用户期望立即生效               |
| 计数变更       | 防抖同步（停止操作3秒后）| 避免频繁请求，用户点击加减时不会卡顿     |
| 小程序切后台   | 立即同步                 | 保证数据不丢失                           |
| 小程序打开     | 拉取云端数据             | 换设备后能拿到最新数据                   |
| 定时心跳       | 每5分钟检查              | 兜底机制，防止意外丢失                   |

**实现要点：**
- 每个计数器数据增加 `updatedAt` 字段记录最后修改时间
- 本地存储 `lastSyncTime` 记录上次同步时间
- 防抖逻辑：计数操作后启动3秒倒计时，新操作重置倒计时，倒计时结束触发同步

## 六、退出登录与数据合并

### 6.1 退出登录处理

| 数据位置 | 处理方式 |
| -------- | -------- |
| 本地数据 | 保留     |
| 云端数据 | 保留     |

### 6.2 重新登录数据合并

采用时间戳合并策略：

```
重新登录时：
1. 拉取云端数据
2. 对比本地数据时间戳 vs 云端数据时间戳
3. 对于每个计数器：
   - 本地有、云端无 → 上传到云端
   - 本地无、云端有 → 下载到本地
   - 两边都有 → 取最新时间戳的数据
```

| 数据类型       | 合并策略                                       |
| -------------- | ---------------------------------------------- |
| 计数器列表     | 取并集（合并所有计数器）                       |
| 单个计数器数据 | 按 `updatedAt` 时间戳，取最新的                |
| 图解数据（会员）| 云端优先（因为可能有其他设备更新）             |

### 6.3 边界情况处理

| 场景                         | 处理方式                                       |
| ---------------------------- | ---------------------------------------------- |
| 退出后本地新增计数器         | 重新登录时上传到云端                           |
| 退出后本地修改计数           | 按时间戳合并，本地更新则保留本地               |
| 退出后在其他设备删除计数器   | 云端数据覆盖本地，删除本地已删除的             |
| 超限数据（如>3个计数器）     | 老用户保留；新用户按当前限制处理               |
